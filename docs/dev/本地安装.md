# 本地安装方式

支持两种本地安装方式：基于源码开发或基于构建产物。

## 方式一：基于源码（开发模式）

在不发布到 npm 仓库的情况下，通过以下方式实现 `ccr` 命令的全局可用。

### 方法 A：npm link（推荐）

```bash
# 1. 构建
pnpm build

# 2. 全局链接
npm link
```

完成后 `ccr` 命令可在任意目录使用。`npm link` 会正确处理 `package.json` 中的 `bin` 字段，在全局生成 `ccr` 命令的符号链接。

> **注意**: 推荐使用 `npm link` 而非 `pnpm link --global`。pnpm link 在某些环境下只会创建模块链接但不会生成 bin 链接，导致 `ccr` 命令不可用。

### 方法 B：npm install 本地路径

```bash
# 直接从本地路径安装（会创建全局链接）
npm install -g /path/to/claude-code-router

# 或在项目中本地引用
npm install /path/to/claude-code-router
```

这种方式会将本地仓库作为一个真正的 npm 包安装，自动处理 bin 链接。修改源码后重新构建，下次全局调用时会使用最新产物。

### 开发工作流

修改源码后重新构建即可，全局链接自动指向最新产物，无需再次 link：

```bash
pnpm build
```

对于 `npm install -g /path` 的方式，需重新安装才能获取最新构建。

### 取消全局链接

```bash
npm unlink -g
```

## 方式二：基于构建产物

构建完成后，可直接使用 `dist/cli.js` 执行，或安装到全局。

### 临时使用（无需安装）

```bash
# 构建后直接运行
node /path/to/repo/dist/cli.js --help
```

### 全局安装

```bash
# 构建
pnpm build

# 复制可执行文件到全局 bin 目录
mkdir -p ~/.local/bin
cp dist/cli.js ~/.local/bin/ccr
chmod +x ~/.local/bin/ccr

# 确保 ~/.local/bin 在 PATH 中
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
```

> **注意**: 与 `npm link` 不同，基于构建产物的方式不会自动更新。每次重新构建后需重新复制文件。
